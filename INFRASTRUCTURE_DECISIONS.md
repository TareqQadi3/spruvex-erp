# قرارات البنية التشغيلية للإطلاق التجريبي | Infrastructure Decisions

> آخر تحديث: 2026-07-10 — قائمة قرارات قبل دفع أي تكاليف. لا تغييرات كود.

---

## الحقيقة الحاسمة التي تحكم كل ما يلي

**Neon (قاعدة بياناتنا) لا تملك أي منطقة خليجية** — مناطقها الحالية: أمريكا، فرانكفورت، لندن، سنغافورة، سيدني، ساو باولو (تم التحقق من وثائقها الرسمية بتاريخ اليوم). أقرب منطقة مناسبة لنا: **فرانكفورت**.

وهذا يقلب المنطق البديهي "استضف قرب المستخدم":

- كل طلب API ينفّذ **عدة استعلامات** قاعدة بيانات (صلاحيات + حالة اشتراك + الاستعلام الفعلي).
- API في الخليج + قاعدة في فرانكفورت = كل استعلام يعبر ~90-120ms × عدد الاستعلامات = **200-500ms+ لكل طلب**.
- API بجوار القاعدة في ألمانيا = استعلامات بـ1-2ms، والمستخدم السعودي يدفع رحلة واحدة فقط ~100-130ms = **~100-150ms إجمالاً لكل طلب**. أسرع بوضوح.
- الملفات الثابتة (واجهة POS والموقع) تُخدَم من حواف Cloudflare **داخل السعودية** — تحميل الصفحات يبقى محلياً بالكامل.

> البديل "كل شيء في الخليج" يتطلب التخلي عن Neon (استضافة Postgres ذاتية تُفقدنا PITR والإدارة، أو RDS في البحرين بتكلفة وتعقيد أعلى — وحتى AWS Lightsail البسيط **غير متوفر** في البحرين، تم التحقق). القرار: يُؤجَّل لما بعد التجربة، ويُتخذ ببيانات قياس فعلية لا تخميناً.

## 1) خيار الـVPS — التوصية: Hetzner (فالكنشتاين/نورمبرغ، ألمانيا)

| المعيار | **Hetzner CX32 (الموصى به)** | Hetzner CX22 | DigitalOcean (فرانكفورت) | خيار خليجي (EC2/Azure/Oracle) |
|---|---|---|---|---|
| السعر/شهر | **~€6.80** (4 vCPU/8GB/80GB) | ~€3.79 (2 vCPU/4GB/40GB) | ~$24-28 لمواصفات مماثلة | أعلى + تعقيد إداري أكبر |
| الأداء | ممتاز سعرياً (أفضل price/perf في السوق) | كافٍ لكنه ضيق | جيد | جيد |
| قرب قاعدة البيانات | **~1-2ms إلى Neon فرانكفورت** | نفسه | نفسه تقريباً | ~90-120ms لكل استعلام ❌ |
| سهولة الإدارة | لوحة بسيطة، snapshots، firewall مدمج | نفسه | أسهل قليلاً | Console معقّدة للمهمة |
| التوسّع | ترقية بنقرة (حتى CX52)، أو إضافة خادم ثانٍ لاحقاً | نفسه | نفسه | نعم لكن بكلفة |
| النقل الشهري | 20TB مشمولة | 20TB | 1-4TB ثم رسوم | رسوم نقل من أول بايت |

**لماذا CX32 وليس CX22؟** الحمولة الكاملة (nginx + api + worker + redis + واجهتان ثابتتان + Docker) على 4GB ضيقة؛ فارق €3/شهر يشتري راحة 8GB وعدم القلق من OOM أثناء أول تجربة حقيقية. (+ ~€0.5/شهر لعنوان IPv4 الأساسي.)

## 2) مزود النطاق — التوصية: Cloudflare Registrar

| المعيار | **Cloudflare (الموصى به)** | Namecheap / Porkbun |
|---|---|---|
| السعر | بسعر التكلفة (~$10-11/سنة لـ.com، بلا رفع تجديد) | ~$10-15/سنة مع عروض أول سنة |
| DNS | الأفضل عالمياً (anycast، أسرع انتشار) — وهو نفسه CDN وWAF | جيد |
| إدارة SSL | **تُغني عن certbot كلياً**: شهادة Origin CA صالحة 15 سنة تُركَّب في nginx مرة واحدة، وشهادات الحافة تلقائية للأبد (وضع Full Strict) | تحتاج certbot + تجديد |
| إضافات مجانية | CDN من حواف السعودية، حماية DDoS، R2 (تخزين 10GB مجاني لنسخ pg_dump) | لا |

> هذا يبسّط الخطوة 2 (SSL) في `PILOT_LAUNCH_PLAN.md`: بدلاً من certbot + cron تجديد → شهادة Origin CA واحدة تدوم 15 سنة.

**نطاق .sa / .com.sa**: يتطلب سجلاً تجارياً/علامة سعودية عبر مسجّلي SaudiNIC — **يُؤجَّل** حتى وجود الكيان الرسمي. ابدأ بـ.com الآن (فوري)، وأضف .sa لاحقاً كإعادة توجيه/هوية.

## 3) البنية النهائية للإطلاق التجريبي

```
المستخدم (السعودية/الخليج)
        │
        ▼
┌─ Cloudflare (مجاني) ──────────────────────────────┐
│  Registrar + DNS + CDN حواف السعودية + DDoS + WAF │
│  TLS الحافة تلقائي · Origin CA cert (15 سنة)       │
└──────────────┬─────────────────────────────────────┘
               │ (Full Strict TLS)
               ▼
┌─ Hetzner CX32 — فالكنشتاين، ألمانيا ──────────────┐
│  docker compose --profile full                     │
│  nginx ─┬→ api (Express)  ←→ redis                 │
│         ├→ worker (BullMQ) ↗                       │
│         ├→ pos-system (static)                     │
│         └→ spruvex-site (static)                   │
└──────────────┬─────────────────────────────────────┘
               │ ~1-2ms
               ▼
   Neon Postgres (Launch) — فرانكفورت
   PITR + فرع منفصل عن التطوير كلياً
```

- **المراقبة**: UptimeRobot (مجاني) على `/readyz` + Sentry (خطة مجانية) في `errorHandler.ts` (تُدمج ضمن خطوات النشر — commit صغير واحد).
- **النسخ الاحتياطي**: Neon PITR + `pg_dump` أسبوعي عبر cron → `rclone` إلى Cloudflare R2 (ضمن الـ10GB المجانية — نفس حساب Cloudflare، بلا مزوّد إضافي).

## 4) ماذا نشتري الآن فعلاً، وماذا نؤجل

### نشتري الآن (المجموع: **~$30/شهر + ~$11/سنة**)

| البند | التكلفة | ملاحظة |
|---|---|---|
| نطاق .com (Cloudflare) | ~$10-11/سنة | القرار الوحيد الذي يحتاج اختيار اسم |
| Hetzner CX32 + IPv4 | ~€7.3/شهر | فالكنشتاين أو نورمبرغ |
| Neon خطة Launch | ~$19/شهر | تحقق من السعر الحالي عند الاشتراك — التسعير قد يتغير |

### مجاني الآن (يُفعَّل بلا دفع)

Cloudflare (DNS/CDN/Origin CA/R2) · UptimeRobot · Sentry (الخطة المجانية) · GitHub Actions (ضمن الحد المجاني).

### يُؤجَّل صراحة (لا تدفع ولا تُنشئ حسابات)

| البند | متى يُتخذ القرار |
|---|---|
| نطاق .sa/.com.sa | بعد السجل التجاري |
| نقل البنية للخليج | فقط إن أثبت القياس أثناء التجربة أن الزمن يؤذي فعلاً |
| خدمة بريد (transactional email) | لا يوجد أي إرسال بريد في الكود أصلاً — عند بناء أول ميزة تحتاجه |
| تخزين ملفات (R2/S3 مدفوع) | عند بناء backend الرفع (غير موجود بتصميم متعمد) |
| Container registry | النشر يبني على الخادم مباشرة؛ يلزم فقط عند تعدد الخوادم |
| خطط مدفوعة للمراقبة/السجلات | عند تجاوز الحدود المجانية فعلياً |
| حسابات مطوّر (سلة/زد/Shopify/تابي/تمارا/Moyasar) | بعد الموقع المنشور والكيان الرسمي — قرار قائم |
